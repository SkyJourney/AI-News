## 结论

**采用 Quartz 5**。

Quartz 5 在 8 项评估维度上全面领先（38/40），且**原生支持 Obsidian Bases**——vault 里已有的 4 个 `.base` 文件 0 改动直接渲染 5 视图 tab。这是 F1 之后 vault 内容自包含化的天然延伸，且 F2.1-F2.7 中约 60% 的"必备+加分"功能（wikilink / backlinks / graph / search / TOC / dark mode / reader mode / OG image / RSS）直接由 Quartz 内置，节省至少 2 天开发。

## 各候选独有王牌 vs 死穴

### Quartz 5

**王牌**：
- **原生 Obsidian Bases 1.9+ 支持**——4 个 `.base` 文件 0 改动，渲染 5 视图 tab（表格/卡片/时间线/异常表）
- **wikilink 4 种形态全解析**：`[[foo]]` / `[[foo|alias]]` / `[[foo#heading]]` / `[[dir/foo]]`
- **46 官方插件即开即用**：backlinks / graph / search / og-image / rss / sitemap / callouts / mermaid / latex / dark mode / reader mode / TOC / explorer 侧边栏
- **Docker Support 已在官方 Dockerfile**（README 里有 hosting 章节）
- 支持 `.canvas` 文件（Obsidian Canvas）

**死穴**：
- Node 22+ 依赖；部署镜像更大（vs Hugo 单二进制）
- LaTeX 插件把中文 `$500M` 误认数学模式产生大量 warning（非阻塞）

### Astro

**王牌**：
- **图像自动优化**：61 图 webp 化，平均省 60%+ 体积
- Build 5s，快于 Quartz
- Astro Content Collections + Zod 有强类型加持（如果我们愿意维护 schema）
- React/Vue/Svelte 组件可插拔

**死穴**：
- 所有 vault 特性都要**自建**：wikilink slug 映射表、backlinks、Bases、Graph、Search、TOC
- Zod schema 与 vault 现有 YAML 冲突（日期字段被自动转 Date）需宽松处理
- POC 里 wikilinks 100% 判定 broken（URL 与实际路径不匹配）

### Hugo

**王牌**：
- **极速构建**：49ms 173 页（Quartz 12s / Astro 5s）
- 单二进制部署，Docker 镜像最小
- 生产环境成熟，SEO/RSS/sitemap 内置
- Go 模板性能强

**死穴**：
- **`[[wikilink]]` 不解析**：Hugo 0.163.3 里 `[markup.goldmark.extensions.wikilink]` 配置未生效，需 pre-processing pipeline 或 Hugo Module（工作量大）
- **不跟随目录 symlink**：必须用 `module.mounts` 手动挂载每个 vault 子目录
- Bases 完全无支持
- Graph View / Search / Explorer 全部自建

## 副产品发现（POC 意外收益）

1. **news-originalizer.md YAML bug**：`original_title` 含冒号未加引号 → 严格 YAML 解析报错。vault 81 md 中 1 个中招（Pessimism's Paradox 那条），本次 POC 已手工修复。**F1 收尾需在 news-originalizer.md 加"含冒号强制引号"约束**，避免下次 F1 真跑再出。
2. **中文 `$` 与 LaTeX 冲突**：`$500M` / `$100 亿` 被 Quartz LaTeX 插件误认数学模式，产生大量 warning——决定关 latex 插件或配置更严的 delimiter。
3. **Bases 是 Quartz 5 的差异化优势**——如果未来选择 Astro/Hugo，需要重写 Base 查询与视图渲染逻辑（非小事，~1 天工作量）。
4. **Hugo 不跟随目录 symlink**——非 Quartz/Astro 共有的行为（都跟随），Hugo 需要显式 `module.mounts`。
